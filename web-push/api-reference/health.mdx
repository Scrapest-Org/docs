---
title: "Health Check"
description: "Check if the Web Push Service is operational"
---

This endpoint allows you to verify that the Web Push Service is running correctly.

<ResponseField name="message" type="string">
  A confirmation message that the service is running.
</ResponseField>

<ResponseField name="status" type="object">
  Detailed status of internal components.
  <Expandable title="properties">
    <ResponseField name="service_running" type="boolean">
      Indicates if the check-in interval is active.
    </ResponseField>
    <ResponseField name="ws_connected" type="boolean">
      Status of the WebSocket connection to Autopush.
    </ResponseField>
    <ResponseField name="twitter_authenticated" type="boolean">
      Status of the Twitter session authentication.
    </ResponseField>
    <ResponseField name="push_registered" type="boolean">
      Indicates if the push service is fully synchronized.
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
  ```bash Example Request curl -X GET http://localhost:6969/ ```
</RequestExample>

<ResponseExample>
  ```json Response
  {
    "message": "Web Push Service is running",
    "status": {
      "service_running": true,
      "ws_connected": true,
      "twitter_authenticated": true,
      "push_registered": true
    }
  }
  ```
</ResponseExample>
