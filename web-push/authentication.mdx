Most endpoints in the Web Push Service are protected and require authentication.

## API Token

To authenticate your requests, you must include the `x-api-token` header with your configured secret key.

<ParamField header="x-api-token" type="string" required>
  The secret API key configured in your `WEBPUSH_KEY` environment variable.
</ParamField>

## Live Demo

You can interact with our live Web Push instance at:

`https://webpush-production.up.railway.app/`

<Note>
  Authentication is required for all state-changing operations and sensitive
  data access on the live instance.
</Note>

````
### Error Responses

If the `x-api-token` is missing or invalid, the API will return a `400 Bad Request` error.

{
  "status": "error",
  "error": "Invalid auth token provided"
}
````
