Most endpoints in the Web Push Service are protected and require authentication.

## API Token

To authenticate your requests, you must include the `x-api-token` header with your configured secret key.

<ParamField header="x-api-token" type="string" required>
  The secret API key configured in your `WEBPUSH_KEY` environment variable.
</ParamField>

## Live Demo

You can interact with our live Web Push instance at:

`https://webpush-production.up.railway.app/`

<Note>
  Authentication is required for all state-changing operations and sensitive
  data access on the live instance.
</Note>

### Example Request

<CodeGroup>

```bash curl
curl -X GET https://webpush-production.up.railway.app/health \
  -H "x-api-token: your-secret-key"
```

```javascript fetch
fetch("https://webpush-production.up.railway.app/health", {
  method: "GET",
  headers: {
    "x-api-token": "your-secret-key",
  },
})
  .then((response) => response.json())
  .then((data) => console.log(data));
```

```python python
import requests

url = "https://webpush-production.up.railway.app/health"
headers = {
    "x-api-token": "your-secret-key"
}

response = requests.get(url, headers=headers)
print(response.json())
```

```go go
package main

import (
	"fmt"
	"io/ioutil"
	"net/http"
)

func main() {
	url := "https://webpush-production.up.railway.app/health"
	method := "GET"

	client := &http.Client{}
	req, err := http.NewRequest(method, url, nil)

	if err != nil {
		fmt.Println(err)
		return
	}
	req.Header.Add("x-api-token", "your-secret-key")

	res, err := client.Do(req)
	if err != nil {
		fmt.Println(err)
		return
	}
	defer res.Body.Close()

	body, err := ioutil.ReadAll(res.Body)
	if err != nil {
		fmt.Println(err)
		return
	}
	fmt.Println(string(body))
}
```

```rust rust
use reqwest::header;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = reqwest::Client::new();

    let res = client
        .get("https://webpush-production.up.railway.app/health")
        .header("x-api-token", "your-secret-key")
        .send()
        .await?
        .text()
        .await?;

    println!("{}", res);
    Ok(())
}
```

</CodeGroup>

### Error Responses

If the `x-api-token` is missing or invalid, the API will return a `400 Bad Request` error.

```json 400 Bad Request
{
  "status": "error",
  "error": "Invalid auth token provided"
}
```
